## Quickstart

Get up and running in under a minute. Every primitive ships in one binary — no Kafka, no Redis, no separate workflow engine.

<Tabs>
  <Tab title="macOS">

```bash
# Install via Homebrew
brew install floruntime/tap/flo

# Start the runtime
flo server start

# Store and read a value
flo kv set hello "world"
flo kv get hello
```

  </Tab>
  <Tab title="Linux">

```bash
# Download and install
curl -fsSL https://get.floruntime.io | sh

# Start the runtime
flo server start

# Create a stream and append an event
flo stream create events
flo stream append events '{"type":"login","user":"alice"}'
```

  </Tab>
  <Tab title="Docker">

```bash
# Pull and run
docker run -p 4222:4222 -p 9090:9090 floruntime/flo:latest

# Use the CLI from another terminal
flo kv set greeting "hello from docker"
flo kv get greeting
```

  </Tab>
</Tabs>

## Primitives

Four data primitives — all backed by the same Raft-replicated log, shaped to fit different access patterns.

<CardGroup cols={2}>
  <Card title="KV" href="/primitives/kv">Hash-indexed key-value store with MVCC, TTL, CAS, and prefix scans.</Card>
  <Card title="Streams" href="/primitives/streams">Partitioned append-only logs with consumer groups and exactly-once delivery.</Card>
  <Card title="Queues" href="/primitives/queues">Priority queues with lease-based delivery, dead-letter support, and visibility timeouts.</Card>
  <Card title="Time-Series" href="/primitives/time-series">Columnar storage with InfluxDB line protocol ingest and the FloQL query language.</Card>
